## What I do

- Write and improve `README.md` and other markdown documentation
- Write Rust doc comments (`///`) for public items in `dantzigrs` and `py-dantzigpy`
- Write Python docstrings following the Google style convention
- Keep the `.opencode/agents/agents.md` setup file accurate and up to date
- Ensure build instructions and usage examples are correct and tested

## When to use me

Use this skill when:
- Adding or updating documentation for a new function, module, or crate
- Reviewing or improving existing doc comments or README sections
- Writing usage examples that will appear in public documentation
- Updating the project roadmap or architecture description

## Style guidelines

- Rust doc comments use `///` and follow standard rustdoc conventions
- Python docstrings use Google style (Args/Returns/Raises sections)
- Keep README code examples runnable and tested
- Prefer concrete examples over abstract descriptions
- Document the "why", not just the "what"
